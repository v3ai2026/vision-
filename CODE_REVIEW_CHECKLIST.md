# ✅ Code Review 快速检查清单

## 提交者自检 (开发者在提交 PR 前)

### 基础检查
- [ ] 代码通过 `npm run lint` 检查
- [ ] 代码通过 `npm run type-check` 检查
- [ ] 代码通过 `npm run format` 格式化
- [ ] 所有测试用例通过 `npm run test`
- [ ] 本地运行正常 `npm run dev`

### 代码质量
- [ ] 变量、函数、组件命名清晰且符合规范
- [ ] 所有 Props 和函数参数有 TypeScript 类型定义
- [ ] 复杂逻辑有注释说明
- [ ] 没有 `console.log`、`debugger` 等调试代码
- [ ] 没有注释掉的代码
- [ ] 没有使用 `any` 类型（除非必要）

### 功能完整性
- [ ] 实现了所有需求功能
- [ ] 处理了所有边界情况
- [ ] API 调用有完整的错误处理
- [ ] 加载状态处理完善
- [ ] 错误提示用户友好

### 性能优化
- [ ] 使用 `computed` 缓存计算属性
- [ ] 大列表使用虚拟滚动
- [ ] 图片使用懒加载
- [ ] 避免不必要的组件重新渲染

### 安全性
- [ ] 用户输入经过验证和转义
- [ ] 避免使用 `v-html` 或已使用 DOMPurify
- [ ] API 请求包含必要的认证信息
- [ ] 敏感数据不暴露在前端

## 审查者检查 (Code Reviewer)

### 代码逻辑
- [ ] 代码逻辑正确且易于理解
- [ ] 没有明显的 Bug 或逻辑错误
- [ ] 边界情况处理完善
- [ ] 错误处理机制完整

### 代码规范
- [ ] 符合项目编码规范
- [ ] 命名准确且一致
- [ ] 代码结构清晰
- [ ] 没有重复代码（DRY 原则）

### 架构设计
- [ ] 组件职责单一（单一职责原则）
- [ ] 模块划分合理（高内聚低耦合）
- [ ] 状态管理规范
- [ ] 可复用性良好

### 性能安全
- [ ] 没有明显的性能问题
- [ ] 没有内存泄漏风险
- [ ] 没有安全漏洞
- [ ] 资源加载优化合理

### 可维护性
- [ ] 代码易于理解和维护
- [ ] 注释和文档充分
- [ ] 测试覆盖充分
- [ ] 便于未来扩展

## 评论模板

### Blocker（必须修改）
```
[blocker] 这里会导致内存泄漏，组件销毁时需要清理定时器：

\`\`\`typescript
onUnmounted(() => {
  clearInterval(timer);
});
\`\`\`
```

### Question（需要澄清）
```
[question] 这个函数的作用是什么？能否添加注释说明？
```

### Optional（建议优化）
```
[optional] 这里可以使用 `computed` 缓存计算结果，避免重复计算：

\`\`\`typescript
const filteredList = computed(() => {
  return list.value.filter(item => item.active);
});
\`\`\`
```

### Praise（正面评价）
```
✨ 这个错误处理写得很完善！
👍 类型定义很清晰，赞！
🎯 这个性能优化思路不错！
```
